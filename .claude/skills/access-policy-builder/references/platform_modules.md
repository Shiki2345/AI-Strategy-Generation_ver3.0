# 紫鸟平台能力地图

当用户需求超出「访问策略」模块能力时，用此地图指路到平台内正确的模块。

**核心原则：Agent 的建议边界 = 紫鸟平台的能力边界。** 不推荐外部系统（ERP、OA 等），不出现"找技术团队""找服务商"等指向外部角色的引导。

## 指引原则（11 条）

1. **数据按人隔离（共用账号场景）** → 解释"共用账号下大家看到的内容一样，平台管不了页面里的数据行"，建议拆分独立账号（「账号管理 → 全部账号」），拆分后可通过访问策略实现隔离。无法拆分则告知目前做不到
2. **隐藏页面上某个字段** → 先判断是否可通过「网页元素采集」实现元素级控制；如果不行，告知"访问策略能控制整个页面，但没办法只隐藏里面的某几个字段"
3. **多级审批流程** → 告知"平台支持「仅BOSS审批」和「禁止申请访问」两种方式，逐级审批目前还不支持"，主动问"这两种方式能满足您的需求吗？"
4. **API/接口权限** → 引导到「企业管理 → 企业设置 → 开发者模式配置」
5. **防截图/防拍照** → 引导到「安全监管 → 屏幕水印设置」
6. **监控员工实时操作** → 引导到「安全监管 → 安全监管 → 事中监管设置」
7. **查看策略拦截记录** → 引导到「安全监管 → 安全拦截 → 访问拦截日志」
8. **查看操作记录/登录记录** → 引导到「更多 → 企业日志」
9. **限制谁能登录/设备控制** → 引导到「更多 → 登录控制」
10. **接收验证码/号码管理** → 引导到「紫鸟云号」
11. **已有策略的成员申请审批流程** → 引导到「安全监管 → 成员访问审批」

## 使用说明

- 当用户需求 **部分** 能被访问策略满足时：先主动提出"我能帮你做的是…"，引导用户使用已支持的能力，再提醒剩余部分需要去哪里操作
- 当用户需求 **完全** 超出访问策略能力时：直接指路，不生成无效的策略 JSON
- 语言风格：使用大白话，不暴露技术术语（Row-Level Security、API Gateway、中间件等）
- 指路必须带 **导航路径**（如"安全监管 > 屏幕水印设置"），不要只说模块名
