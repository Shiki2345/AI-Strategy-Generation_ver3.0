# AI 辅助策略生成模块

> 版本：v2.3 · 更新日期：2026-04-22
> 交付形式：VSCode 工程目录（含 Claude Code Skill）

---

## 项目简介

本项目为紫鸟浏览器平台构建了一套 **AI 辅助访问策略生成系统**。用户用自然语言描述需求（如"禁止客服访问广告页面"），AI 通过多轮对话收集参数，输出可直接填入平台「新增访问策略」表单的 JSON 配置。

核心思路：将访问策略配置领域的专家经验编码为结构化的 Skill（提示词工程制品），让 LLM 每次面对同类问题时都能按专家水准执行，而不是靠自己即兴发挥。

## 快速开始

### 1. 环境准备

- 安装 [VSCode](https://code.visualstudio.com/)
- 安装 VSCode 的 **Claude Code 插件**（扩展市场搜索 "Claude Code"）
- 登录 Claude 账号

### 2. 打开项目

解压交付包，用 VSCode 打开根目录 `AI辅助策略生成模块/`。

Claude Code 插件启动后会自动加载 `.claude/skills/access-policy-builder/`，识别 `SKILL.md` 中的触发关键词。

### 3. 开始对话体验

在 Claude Code 对话框中直接输入访问策略相关的需求，例如：

```
禁止客服访问广告页面
```

Skill 会自动触发，进入 5 阶段对话流程（意图识别 → 多轮收参 → 确认校验 → 生成 JSON → 交付）。

更多体验示例见 [测试案例.md](测试案例.md)（11 个案例，覆盖话题边界、正常流程、条件支持、不支持场景）。详细使用说明请查阅外部操作说明书。

## 当前 demo 的能力

1. **网页黑名单**（已限制模式）— 禁止打开指定页面
2. **网页行为管控**（可访问模式）— 允许打开但限制页面内的复制/下载/打印/键盘输入/查看密码框等 7 项操作
3. **审批控制** — 仅 boss 可审批 / 完全禁止申请访问
4. **生效范围控制** — 按成员/部门/角色、按账号、按时段生效

> 当前 demo 支持网页黑名单 + 页面内行为管控（只读模式）+ 审批控制 + 按人员/账号/时段的生效范围控制，不支持白名单和元素级限制。

## 系统工作流程

```
用户自然语言描述
  │
  ▼
Phase 1：意图识别 + 能力边界检查
  │  判断需求是否在平台能力范围内
  │  不支持 → 坦诚告知 + 推荐平台内替代方案
  │  条件支持 → 先问前提条件（如账号结构）
  │  支持 → 继续
  ▼
Phase 2：多轮对话收集参数
  │  URL 知识库检索（219 条 Amazon 页面）
  │  逐项收集：生效成员、账号、时段、URL 规则、审批限制
  ▼
Phase 3：确认汇总
  │  校验策略名称 ≤24 字符、描述 ≤400 字符、域名精准度
  ▼
Phase 4：生成表单输出
  │  输出 JSON，字段名与平台表单标签一一对应
  ▼
Phase 5：交付
```

## v2.3 能力评估

### 能做好的场景

| 场景 | 覆盖率 | 说明 |
|------|--------|------|
| 角色权限批量配置 | ~85% | 6 种角色模板直接输出；自定义角色由 LLM 从模块映射表组合 |
| 单页面屏蔽/审批 | ~70% | 目标页面在 219 条 URL 中时表现良好 |
| 时间条件策略 | ~90% | 预设完善，常见时段直接套用 |
| 不可实现需求的拦截 | ~80% | 能力边界 + 平台地图能识别并坦诚告知，不编造配置 |

### 有风险的场景

| 场景 | 覆盖率 | 风险点 |
|------|--------|--------|
| 数据隔离 | ~50% | 需先确认账号结构（独立/共用），用户自己不清楚时对话会卡住 |
| 复合需求 | ~40% | "工作时间可看不可导出，下班后完全禁止"需拆多条策略，当前对话面向单条策略设计 |
| 非标页面 | ~30% | 用户描述的页面不在 219 条中，且无法提供 URL 时，无法生成准确 URL 规则 |

### 系统层面限制（Skill 提示词无法解决）

1. **不知道用户实际环境**：用户公司的部门、角色、账号需用户自行确认
2. **从 JSON 到填表仍是手动**：输出 JSON 后需对照逐项填写
3. **没有部署后反馈**：策略生效后若屏蔽错页面，系统无法感知和排查

### 总体判断

v2.3 把系统从"框架完善但数据缺失"推进到了"框架和数据都就绪"。对常见的单条策略配置需求（占实际需求大部分），系统已经能输出正确、可直接填表的 JSON。它是一个有效的**配置辅助工具**——把"配置访问策略"的门槛从"需要理解平台表单的每个字段和选项"降低到"用自然语言说清楚你想限制谁访问什么"。

剩余限制主要在系统与平台之间的集成层面（不知道环境、不能自动填表），这些是 Skill 提示词层面无法解决的。
